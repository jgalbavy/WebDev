CSS Parts

Two Parts: Selector and Declariation
Selectors indicate which element the rule applies to. The same rule can apply to more than one element if you separate the element names with commas.
Declarations indicate how the elements referred to in the selector should be styled. Declarations are split into two parts (a property and value) seperated by a colon.

CSS selectors

Universal selects all elements = *
Type Selector: matches element names = h1, h2, h3 {}
Class selector: matches an alemenet whose class attriubte has a valye that matches the one specified after the . = .class {}
ID Selector: Matches whose id attribute matches the one specified = #id {}
Child selector: matches an element that is a direct child = li>a {} ---> targets any *a* elelement that are children of an *li* element (but not other a ele.)
Descendent selector: matches an ele. that is a descendant of another (not JUST direct) = p a {} --> targets ANY a inside P even if there are elements nested between
Adjacent sibling selector: matches an element that is the next sibling of another = h1+p {}
General Sibling Selector: matches an ele. that is a sibling of another althought it does not have to be direclty preceding element = h1~p {}

How it cascades

Last Rule: latter of two same named rules takes precedence
Specificity: higher specificity takes precendence (h1 more detailed than *)
!Important: marks it as important rule
Cascading is VITAL to elegent pages.
Inheretance: child elements inhertet from parents.
