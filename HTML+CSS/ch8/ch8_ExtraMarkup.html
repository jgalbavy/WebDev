Evolution of HTML

HTML4: Lots has been replaced by CSS
XHTML: Added to include XML use, made the language more strict
HTML5: In progress. Dont need closing tags, New elements / attributes.

Doctypes. First line of file include the doctype, html5 by default

Comments: "!-- --" where " = angled brackets

ID Attribute: every HTML element can carry one and its used to uniquely identify that element from others.
Make it unique, duh or it wont be unique. Very useful for CSS and JS.

Class attribute: Used to uniquely identify a group of elements (Like classes in other languages), Couple this with CSS for ideal presentation.

Block elements: Block level elements always start on a new line.
Examples: h1, p, ul, li

Inline elements: Elements always appear to continue on the same line.
Examples: a, b, em, img

Grouping Text and Elements in a block: "div" element.

Grouping text and elements in line: "span" element. Combo it with a span class to make in line modifications.

IFrames is like a little window cut into the page to see another page. Short for inline frame. Common use to to embed google maps.
Attributes:
Src - specifies the URL to show in the iframe.
height and width - the usual
scrolling - Not supported in HTML5, Whether it should have scrollbars or not.
Frameborder - Not supported in HTML5. true or false for border.
seamless - New to 5, doesnt need a value.

Information about your page

"Meta" lives inside the "head" element and contains info about the page. Not visible to users but VERY useful (like telling search enginers about the page).
Empty element so no closing tag. Attributes are:
name, content used together
Description- used by search enginers to understand the page
Keywords - list of words that a user might search for on to find the page (not used in practice tho)
Robots - indicates whether search enginers should add this page to their search results or not
Author - you?
Pragma - prevents the browser from caching the page (loading locally)
Expires - should itand when
